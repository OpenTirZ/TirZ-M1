# Feed Forward Network (FFN)

![Feed Forward Network](image.png)

## Overview

This module implements the **Feed Forward Network (FFN)** used within each Transformer block of the TirZ M1 architecture.

While the Multi-Head Attention mechanism allows tokens to exchange information with one another, the Feed Forward Network processes each token independently and learns complex non-linear transformations of the token representations.

The FFN is one of the primary sources of model capacity in Transformer architectures.

---

## Architecture

The TirZ M1 Feed Forward Network consists of:

```text
Embedding Dimension
        │
        ▼
Linear Layer
(emb_dim → 4 × emb_dim)
        │
        ▼
GELU Activation
        │
        ▼
Linear Layer
(4 × emb_dim → 8 × emb_dim)
        │
        ▼
GELU Activation
        │
        ▼
Linear Layer
(8 × emb_dim → emb_dim)
        │
        ▼
Output
```

Unlike the standard Transformer FFN, which typically uses a single expansion layer, TirZ M1 employs a deeper expansion strategy to increase representational power.

---

## Implementation

```python
class FeedForward(nn.Module):
    def __init__(self, cfg):
        super().__init__()

        self.layers = nn.Sequential(
            nn.Linear(cfg["emb_dim"], 4 * cfg["emb_dim"]),
            GELU(),

            nn.Linear(4 * cfg["emb_dim"], 8 * cfg["emb_dim"]),
            GELU(),

            nn.Linear(8 * cfg["emb_dim"], cfg["emb_dim"])
        )

    def forward(self, x):
        return self.layers(x)
```

## Mathematical Representation

Given an input tensor:

```text
X ∈ ℝ^(batch_size × sequence_length × emb_dim)
```

The network computes:

```text
H₁ = GELU(XW₁ + b₁)
```

```text
H₂ = GELU(H₁W₂ + b₂)
```

```text
Output = H₂W₃ + b₃
```

where:

* `W₁`, `W₂`, `W₃` are learnable weight matrices
* `b₁`, `b₂`, `b₃` are learnable bias vectors

## Example Dimensions

Suppose:

```text
emb_dim = 768
```

Then the FFN performs:

```text
768
 ↓
3072
 ↓
6144
 ↓
768
```

This temporary expansion allows the network to learn richer and more expressive representations before compressing them back to the original embedding size.

## Why Use Feed Forward Networks?

Attention determines **which information is important**.

Feed Forward Networks determine **how that information should be transformed**.

Benefits include:

* Learning non-linear relationships
* Increasing model capacity
* Improving representation quality
* Capturing higher-level semantic patterns
* Enhancing language understanding

---

## Why GELU?

The TirZ M1 FFN uses the GELU activation function because it:

* Provides smooth gradients
* Improves optimization stability
* Works exceptionally well in Transformer models
* Preserves useful information better than ReLU

The activation pipeline becomes:

```text
Linear
  │
  ▼
GELU
  │
  ▼
Linear
  │
  ▼
GELU
  │
  ▼
Linear
```

## Position Inside a Transformer Block

```text
Input
  │
  ▼
Multi-Head Attention
  │
  ▼
Add & LayerNorm
  │
  ▼
Feed Forward Network
  │
  ▼
Add & LayerNorm
  │
  ▼
Output
```

Every Transformer block contains one Feed Forward Network.

---

## Learnable Parameters

For an embedding dimension of 768:

```text
Layer 1 : 768  → 3072
Layer 2 : 3072 → 6144
Layer 3 : 6144 → 768
```

This contributes a substantial portion of the Transformer's total trainable parameters.

In many language models, the FFN contains more parameters than the attention mechanism itself.

## Purpose

The Feed Forward Network is responsible for transforming contextual token representations generated by the attention mechanism into richer and more expressive feature spaces.

In TirZ M1, the FFN serves as a powerful non-linear computation module that enhances the model's ability to learn language patterns, semantic relationships, and complex textual structures required for large-scale language modeling.
